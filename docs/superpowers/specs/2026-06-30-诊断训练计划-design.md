## 1. 背景与目标

现有「统计」页有 Phase 5/6 的**事后**个性化:用户先随便练,系统从错题堆(`reviewPile`)反推漏洞(`classifyMiss`)、画像、需练度排序。

本功能加一条**事前**路径:用户首次进来先做**问卷 + 诊断测试**,系统**主动测出**水平与漏洞,生成一份**结构化的 20 天每日训练计划**,每天可执行打卡。两者互补——新功能在前端"建档 + 给节奏",旧功能在后端"边练边修正",共享同一套 `classifyMiss`/leak 数据。

**一句话**:把"不知道该练什么"变成"今天打开就知道练什么"。

### 诚实红线(继承 PRODUCT.md §2/§6,任何实现不得违反)

- 报告与计划全程标注「**vs 参考范围**」,**不冒充 solver 精确解**。
- **不显示编造的 EV 或精确混合频率**。报告里的百分比只是"你这些手的**正答率**"(实测,可算),不是 EV、不是频率。
- **不 claim TAG/LAG** 等带攻击性结论;画像只给"偏松/偏紧、偏被动/偏激进"倾向。
- 样本越少措辞越保守(见 §5)。
- 零新数据:诊断题、计划内容全部复用现有 spot/题库/引擎。

## 3. 问卷(免费,4 题)

每题单选,30 秒填完,每项都直接喂给后面:

| # | 题 | 选项 | 用途 |
|---|---|---|---|
| 1 | 主战场 | 现金 / 锦标赛;深度:深码 100bb · 中浅 20-40bb · 短码 ≤15bb | **诊断与计划聚焦哪些场景**(短码玩家不练 100bb 开局) |
| 2 | 自评水平 | 新手 / 进阶 / 高手 | **仅定诊断初始难度**,测完用真实表现校正(诚实:自评不算数) |
| 3 | 每天能练多久 | 5 / 10 / 20 分钟 | **定每日主题手数** N(见 §6) |
| 4 | 主要目标 | 修我的漏洞 / 系统过一遍 / 专攻某场景 | **定计划编排重心**(见 §6) |

**存储**:`localStorage` → `onboard = {field, depth, level, minutes, goal, ts}`。

**场景聚焦映射**(问卷 → 诊断/计划覆盖哪些 `MODES` 场景):
- 现金深码 → 开局 / 防守 / 面对3bet / 面对4bet / 挤压 / 冷跟
- 锦标赛中浅 → 上述 + 推弃 / 跟全下(call-off)
- 短码 ≤15bb → 推弃 / 跟全下为主,开局防守为辅
- 目标"专攻某场景" → 追加一题选具体场景,诊断/计划侧重它

## 4. 诊断测试(免费)

**方法 A(已选)**:复用现有 `buildSmartQueue` 引擎,不造新题。

- 按问卷聚焦的场景集合,从每个场景的已解锁 spot 抽题,**覆盖均衡**(每场景至少 M 手)。
- 简化版总 **18 手**(各场景 3-4 手,聚焦主战场核心);详细版 **45 手**(全场景铺开 + 每场景更多样本)。
- 出题逻辑沿用 `buildSmartQueue` 的类型均衡 + 难弃牌(`adjacentFolds`),保证不是随机废牌、也不是只考送分题。
- 答题界面**复用现有游戏循环**(`nextHand`/`choose`/`resolve`),但**诊断模式**:不计 HP、不进 `reviewPile` 常规堆,而是把每手的 `{spot, hand, choice, correct}` 收集到诊断结果集。
- 答完 → 对每手跑现有 `classifyMiss(rec)`,按场景 + 错误类型聚合。

**诊断模式开关**:游戏循环加一个 `G.diagMode` 标志(或独立的轻量 runner),与正常训练隔离,不污染生涯统计/HP/错题堆。诊断产生的错题**可选**导入 `reviewPile`(供后续计划的"复习"段用)。

**存储**:`diagnosis = {variant:'simple|full', perScene:{[scene]:{n, correct, leakCounts}}, topLeaks:[...], strengths:[...], ts}`。

## 5. 诊断报告(免费,mockup 屏 1)

版式见 `scratchpad/mockup.html` 屏 1。模块:

1. **一句话总评** + 一句解释:如「偏紧 · 进阶 · 该入池时常弃牌,漏掉价值」。由画像倾向 + 自评/实测档位合成。
2. **各场景表现条形**:每场景一条,显示**正答率 %**(实测,vs 参考范围),绿(≥70)/黄(55-69)/橙(40-54)/红(<40)。
3. **Top 3 漏洞**:正答率最低的 3 个场景 + 错误类型红标(太松/太紧/被动/过激)+ 一句人话解释(模板化,复用 `reasonFor` 风格)。
4. **你的强项**:正答率最高的 1-2 个场景,绿色 chip,给正反馈。
5. **20 天计划概览**:哪几天攻哪个漏洞(见 §6 编排)。
6. **底部 CTA**:`开始 Day 1` + PRO 标(免费→付费转化点)。
7. **注脚**:`基于 N 手诊断 · 全部 vs 参考范围,非 solver 精确解`。

**措辞随题数自适应(诚实)**:
- 简化版(18 手):总评只给**大方向**(偏松/偏紧),条形可显示但加"快速估计"标注,**不**细分到具体某场景断言。
- 详细版(45 手):才敢给场景级漏洞断言 + Top 3。
- 任一场景样本 < 阈值(如 4 手)→ 该条标灰/标"样本不足",不参与 Top 漏洞排序。

百分比显示:**保留具体数字**(用户已确认),但语义是"正答率"而非 EV/频率,文案明确。

## 6. 20 天训练计划(Pro,mockup 屏 2)

### 编排(固定 20 天)

- **前 13 天 · 逐个攻克**:按 Top 漏洞优先级(正答率升序)排队,每个漏洞分 ~3-5 天(漏洞越严重/越多手错,分的天数越多;按比例切 13 天)。
- **后 7 天 · 全场景混合巩固**:`buildSmartQueue` 全场景出题 + 重点回炉前段漏洞。
- **退化情形**:若诊断没有明显漏洞(全部 ≥70%)→ 前 13 天按"正答率从低到高"依次过场景(查漏补缺),后 7 天混合。
- **目标=系统过一遍**:忽略漏洞优先,前 13 天均匀覆盖所有场景。
- **目标=专攻某场景**:该场景占前段大头。

### 每日卡片

- **今日主题** = 当天排定的一个场景(如 "Day 3 · 大盲防守 vs 开局")。
- **主题训练 N 手**:N 由问卷时长定 —— 5min≈10 / 10min≈20 / 20min≈40 手;`buildSmartQueue` 在该主题场景内出题。
- **复习 K 手**:`K = min(5, reviewPile 中相关错题数)`;复用现有 `reviewPile` spaced repetition。
- **进度条**:今日完成 X / (N+K) 手。
- **打卡机制**:练完今日手数 = 标记完成 + streak +1;**纯标记,不锁门**——可跳过、可自由练别的场景、可提前做后面的天。streak 断了重置但不惩罚。
- **本周进度**:已完成 ✓ / 今天 ● / 未来 ○(纯展示)。

### 计划存储

`plan = {createdTs, days:[{idx, scene, nMain, done:false, doneTs, accuracy}], streak, lastDoneDate}`

## 10. 明确排除(YAGNI)

- ❌ 一天拼两个主题(违反"一口吃下",后段混合已有多样性)
- ❌ 强制解锁/锁门(赶人走,违反免费内核柔软)
- ❌ 简化版"升级"补做详细版的数据累加(用户已否;两版独立选)
- ❌ 诊断报告给 EV / 精确混合频率(诚实红线)
- ❌ 云同步进度(离线约束;若做另立架构决策)
- ❌ 计划天数动态可变(已定 20 天固定)

## 11. 验收要点

- 诊断模式不污染 HP / 生涯统计 / 常规错题堆(隔离验证)。
- 报告措辞随题数自适应(18 手不出场景级断言)。
- 计划 20 天编排:漏洞优先 → 退化情形(无漏洞)→ 目标变体,都有合理输出。
- 打卡纯标记、streak 正确累计/重置。
- 全程文案 vs 参考范围、无编造 EV/频率;`npm test` 金快照不受影响(新逻辑不动 `MODES`/范围)。
- 双语完整(中英),数据层保持中文唯一来源。
