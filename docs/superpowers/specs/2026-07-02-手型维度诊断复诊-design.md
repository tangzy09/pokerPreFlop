> 研究产物(2026-07-02,基于 main@6dbfa85 只读研究,未改代码)。
> 动机:竞品差评集中在「报告没用、不说错在哪」;现报告只说「face3b 场景 60%」不可操作,

## 0. 现状结论(设计依赖的事实)

- 诊断结果流:`coachStartDiagnosis` → `coachBuildDiagQueue`(coach.js:586-608)→ `SINKS.diag.settle`(app.js:642-647)把每手 `{sceneKey, t, hand, choice, correct, variant}` 压进 `G.diagResults` → `coachFinishDiagnosis`(coach.js:643-652)→ `coachAggregate`(coach.js:48-79,只有 perScene/topLeaks/strengths)→ `coachSaveDiagnosis` 存 `STORE.coachDiagnosis = {variant, agg, verdict, ts}`。
- **`G.diagResults` 里的 `t` 带 Set(R/C/M),但存进 STORE 的只有 agg(纯数字)——新增字段必须保持可 JSON 序列化**,这是硬约束。
- `classifyMiss(rec)`(app.js:1105-1117)吃 `{t,hand,choice,variant}`,六桶输出,可直接复用于手型族维度。
- `buildSmartQueue`(app.js:335-361)按动作类型分桶、`weak`(错题)优先、每类上限 `SMART_PER_TYPE=12`——加权出题的天然挂点。
- `handKind(h)`(app.js:763-772)是服务 `reasonFor` 文案的 8 桶粗分类,**不动它**,新建独立 `handFamily`。
- 每日训练入口 `coachStartDayTraining`(coach.js:503-548)最终走 `newGame()`;`G.handFilter` 只在 `launch()`(app.js:1014)赋值,coach 日训练继承上一次的值——现状 bug 级松散点,本方案顺手收紧。
- 金快照只覆盖 MODES/PACKS 决策矩阵 → 本方案不动这两者,金快照不受影响。
- `test/i18n.test.js` 扫所有 `L('中文')` 调用点 → 每个新增中文字面量必须同步补 I18N_EN。

## 1. handFamily 手牌族分桶

### 1.1 两级粒度:细族 18 个(长期漏洞分析)+ 粗族 7 个(诊断)

诊断只有 18/45 手,`COACH_MIN_SAMPLE=4` 下细族凑不够样本;错题堆/长期统计跨会话累积,细族才可操作。单函数返回细族,粗族由 `FAMILIES[fine].coarse` 派生,无第二套逻辑。

### 1.2 分桶规则(已用 Node 对全 169 手验证,总数=169 ✓)

放在 **app.js 的 hand helpers 区,`neighborsOf` 之后(约 line 330)**:

```js
// 手牌族(细,18 族)。规则只依赖 RIDX/gap/suited,全 169 手完全划分(测试守恒)。
function handFamily(h){
 if(h.length===2){const i=RIDX[h[0]]; return i<=4?'pp_big' : i<=8?'pp_mid' : 'pp_small';} // TT+/99-66/55-22
 const a=RIDX[h[0]], b=RIDX[h[1]], s=h.endsWith('s'), gap=b-a;
 if(a===0){                                        // Ax
  if(s) return b<=4?'axs_bw' : b<=8?'axs_mid' : 'axs_wheel';   // ATs+ / A6s-A9s / A2s-A5s
  return b<=4?'axo_bw':'axo_lo';                               // ATo+ / A2o-A9o
 }
 if(a<=4 && b<=4) return s?'bw_s':'bw_o';          // 双高张(KQ..JT)
 if(s){
  if(gap<=1) return 'sc';                          // 同花连张 T9s..32s
  if(gap===2) return 'sg';                         // 同花隔张 J9s..42s
  if(a<=2)  return 'kqx_s';                        // K/Q 高弱踢同花 K9s-K2s,Q9s-Q2s
  return 'sx_lo';                                  // 低同花杂 J8s..52s
 }
 if(a<=2 && b>4) return 'kqx_o';                   // K/Q 高弱踢 offsuit
 if(gap<=1) return 'off_conn';                     // offsuit 连张 T9o..32o
 return 'off_lo';                                  // 杂牌
}
```

实测每族成员数(进测试断言):pp_big 5 / pp_mid 4 / pp_small 4 / axs_bw 4 / axs_mid 4 / axs_wheel 4 / axo_bw 4 / axo_lo 8 / bw_s 6 / bw_o 6 / sc 8 / sg 8 / kqx_s 16 / sx_lo 28 / kqx_o 16 / off_conn 8 / off_lo 36。
粗族 7 个:`pp`(13) `ax_s`(12) `ax_o`(12) `bw`(12) `sc`(16) `sx_lo`(44) `off`(60)。

## 2. coachAggregate 扩展

### 2.1 取舍:不做 scene×family 交叉

18 手 ÷ 5 场景 ÷ 7 粗族,交叉桶期望样本 <1。方案 = 两条独立轴 + 实例句子桥接:
- `perFamily`(粗族,全场景聚合)→ 比率断言(n≥阈值才说);
- `topLeaks`(场景轴不变)每条附 `examples`(实打实答错的 2-3 手)→ 实例是事实陈述,n=1 也诚实。

## 4. 计划吃手型 drill

### 4.1 day 结构新增字段

`{…, targetFams:['ax_s','sc']|null, recheck:false, recheckDone:false}`;`coachBuildPlan` 取 famLeaks 前 2 族铺给每天(mixed 日同样带)。选"全局 top2 族"而非按场景配族(样本稀 + buildSmartQueue 类型均衡保证不过拟合)。

### 4.2 buildSmartQueue 挂第三优先级(不加 HANDFILTERS,那是用户可选 UI,错层)

```js
// app.js 纯函数(EXPORTS 加入):weak > 目标手型族 > 其余
function smartOrder(cands, famSet){
 const pri=x=> x.weak?0 : (famSet && famSet.has(famCoarse(x.hand)))?1 : 2;
 return [0,1,2].map(p=>shuffle(cands.filter(x=>pri(x)===p))).reduce((a,b)=>a.concat(b),[]);
}
// buildSmartQueue 内:famSet=G.coachFams? new Set(G.coachFams):null;每类型桶内 smartOrder(...).slice(0,SMART_PER_TYPE)
```

## 5. 复诊闭环

- **第 7/14/20 天(idx 6/13/19)= recheck 日**:当天训练前的前置迷你测(`COACH_RECHECK_HANDS=6`),day 卡顶部插「🩺 迷你复诊 · 6 手 [开始复诊]」;完成后显示对比摘要行。不强制。
- **队列**:`coachBuildDiagQueue(scenes, total, handPred)` 加第三个可选参数(hand 过滤);`coachStartRecheck` 按基线 topLeaks 场景收窄 + famLeaks 族过滤,抽不满则放开。
- **启动**:抽公共 `_coachStartDiagRun(queue, kind, dayIdx)` + `_coachDiagKind` 分流;`SINKS.diag`/nextHand/fbNext 的 diag 分支零改动(完全复用诊断管道)。
- **coachFinishDiagnosis 分流**:kind==='recheck' → `coachFinishRecheck(agg)`,**绝不覆盖基线诊断**;摘录 slim `{dayIdx,ts,n,correct,perScene,perFamily}`(丢 misses)push 进 `diag.history[]`。
- **对比报告** `coachRenderRecheckReport`(~50 行,复用 coach-bar-row):只对基线与本次都 n≥3 的键出 `60% → 82%` 箭头行(行尾标 基线 n0/本次 n1);凑不齐只列计数不下结论;尾注 coachVsRef。
- **达标/加练** `coachApplyRecheck`(纯函数,可单测):`COACH_RECHECK_PASS=0.8`,≥5 手才判定;达标 → 后续天 targetFams 换 famLeaks 下一名;不达标 → 最近 2 个 mixed 日转最弱场景主攻(**总天数恒 20 不延长**)。
