## A. 现金多深度(50bb / 200bb)

### A1. 组织:cash 下加带 group 的 variant(照搬 MTT 先例,不加新 format)

```js
// VARIANTS.cash(现有 key 一律不动)
cash:{
 2:{group:'标准 100bb',label:'单挑 2人',short:'单挑'},
 6:{group:'标准 100bb',label:'6人桌',short:'6人'},
 9:{group:'标准 100bb',label:'9人桌',short:'9人'},
 c6_50:{group:'筹码深度 · 6人桌',label:'50bb',sub:'浅码·3bet或弃',short:'6人50bb',players:6},
 c6_200:{group:'筹码深度 · 6人桌',label:'200bb',sub:'超深·隐含赔率',short:'6人200bb',players:6},
},
```
配套:`tablePlayers` 改 `const v=VARIANTS.cash[variant]; return (v&&v.players)||+variant||6;`(一行,老 key 不变)。**不做新 format**(FORMATS/GAMETYPES/defVariant 全要跟,tag 语义重复)。

### A2. PACKS 形状

`c6_50`/`c6_200` 各一个数组,内混 open+defense+face3b 多 mode spot(cash.6 先例);结构化 heroPos/vilPos 必填。深度的 face3b 放本 variant 内,**不做进 face3b format**(variant 数会爆)。

### A3. 数据来源

全部手搓 curated。推弃 Nash 引擎帮不上(jam-or-fold 模型外)。`src` 写 `'curated 50bb reference(公开知识整理)'`。

### A4. 范围手搓原则(相对 100bb)

**200bb**:隐含赔率↑、domination 代价↑、位置价值↑ —— RFI 变化小(EP 砍最弱 offsuit broadway 补同花连子);BB 防守 call 底部略收但投机牌更坚决,3bet 向同花化移动(少 AJo/KQo 类 offsuit 诈唬);face3b 平跟小对价值↑、4bet 值域更紧、AKo 多平跟;face4b call 域扩大、5bet 全下收到 KK+。
**50bb**:set-mining 赔率↓、3bet-or-fold 倾向、全下门槛↓ —— RFI 砍最弱投机同花牌,offsuit broadway 保值;BB 防守 call 明显收窄、3bet 更线性(TT/AJs/KQs 进 3bet);**face3b 是核心新价值**(OOP 几乎 3bet-or-fold、4bet≈全下承诺 JJ+/AK、小对平跟消失);face4b 退化全下或弃(可不做)。

**条数**:每深度 6max = 5 open + 3 defense + 2 face3b ≈ 10 spot、~22 条范围串。9max 深度不做。

## B. 更多 BB 防守(纯数据零代码,P0)

9max 补 3 个(append 到 PACKS.cash[9]):
1. **BB vs SB**(tier 5)—— 全库唯一 BB 有位置的防守,增量最大;防守极宽+3bet 多。
2. **BB vs CO**(tier 5)—— 与 vs BTN 成晚位梯度,6max 同名 spot 可对照校准。
3. **BB vs UTG**(tier 6)—— 紧端锚点(比 6max vs UTG 再紧一档)。
vs HJ/MP 是插值,不做。可选第 4:SB vs BTN(defense mode,call 域极小、3bet-or-fold 为主,本身是一课)。
手搓原则:vs 更早位 3bet 更极化(价值+A5s 阻断)、call 砍底部;vs SB 两域放宽 20-30%。6max 三张现成表是插值锚点。

## C. Limp pot(2 个新 MODES 条目,闭包全复用)

### C1. MODES

```js
// ACT_LABEL 加:overlimp:['跟跛入池','OVER-LIMP','a-call'], iso:['隔离加注','ISO-RAISE','a-raise'], check:['过牌','CHECK','a-call']
iso:{reCol:'raise',actions:[['fold',…],['call',ACT_LABEL.overlimp],['raise',ACT_LABEL.iso]],
 names:{fold:'弃牌',call:'跟跛',raise:'隔离加注'},
 correct:CORRECT.raiseCall,cell:CELL.raiseCall,freq:FREQ.raiseCall,
 catName:{threebet:'隔离加注',mix:'隔离 / 跟跛(混合)'},
 legend:[['threebet','隔离'],['call','跟跛'],['mix','混合'],['edge-call','边缘'],['fold','弃牌']]},
bbvslimp:{reCol:'raise',actions:[['fold',ACT_LABEL.check],['raise',…]],   // 无弃牌:fold 键显示为「过牌」
 names:{fold:'过牌',raise:'加注'},
 correct:CORRECT.raiseFold,cell:CELL.raiseFold,freq:FREQ.raiseFold,
 catName:{fold:'过牌','edge-raise':'加注 / 过牌(边缘混合)'},
 legend:[['raise','加注'],['edge-raise','边缘'],['fold','过牌']]},
```
classifyMiss 把「该加注却过牌」归 tight——语义成立,可接受。

### C2. MODES 外改动点

- `reasonFor` 加 iso/bbvslimp 两分支 + tr 模板 ~6 keys(iso 原则:线性加强范围、尺寸 3bb+每 limper+1bb、OOP 再+1、目标赶盲注打 HU IP);
- `tableModel` 加分支渲染跛入者 1bb;
- 快照 `test:update`(新 mode 自动进 snap.modes,CATS 不变)。

### C3. 分类学

新 format `limp`(与 coldcall 平级):VARIANTS.limp = btn(BTN vs MP 跛入)/co(CO vs UTG 跛入,再紧一档)/bb(BB vs SB 跛入);defVariant 加 `f==='limp'?'btn'`。
范围:isolate = 该位 RFI 砍最弱 20% 线性加强;跟跛域 = 小对/同花连子/中弱同花 A;BB vs SB 加注域 = SB RFI 价值上半+同花投机施压,其余全过牌。

### C4. 诚实表述(重要)

solver 视角 limp 本身次优 → **面对跛入没有 solver 精确解,只有对弱范围的合理调整**。who 写 `'中位跛入(低注额常见) · 应对参考'`;src 写 `'curated vs-limp reference(跛入非均衡动作,无 solver 精确解;按对弱范围的标准调整整理)'`;About 页补假设说明。

## D. Straddle(= 50bb 换皮,严格排 A-1 后)

UTG straddle 2bb = 等效深度减半 + 尺度按 2bb 计 + straddler 最后行动。业界处理 = 等效浅深度 + 死钱修正。
- cash 下加 group variant `c6_str`(不需新 format/mode);PACKS 从 c6_50 拷贝改文案、范围放宽半档;src 明说 `'≈ 等效50bb深度近似(死钱修正),非独立求解'`;
- `tableModel` 加可选字段 `straddle:'UTG'` 渲染 2bb 前置注(几行);
- BTN straddle 不做(规则各家不一,无可靠参考);
- 可选加分项(不排期):tools/pushfold.js 盲注结构参数化 → straddle 推弃 Nash 真可算升 precise。
