# Poker Preflop Coach — 产品说明书 V1.0

> 本文是**我们这个项目**的产品书，脱胎于外部「Poker AI Coach V3.0」愿景，但**按本项目的真实约束裁剪过**：只收录在「零构建 / 离线 / 无真 solver 数据 / 手搓范围」前提下**能诚实做出来**的部分。
> 配套文档：[CLAUDE.md](CLAUDE.md)（架构与验证）。代码现状见 `gto-trainer.html` + `js/`。

---

## 1. 一句话定位

> 一个**诚实、离线、免费内核**的翻前 GTO 直觉与**个人漏洞**训练器 —— 核心永久免费、Pro 解锁差异化，逐步长成「每个 MTT/现金玩家的私人翻前教练」。

护城河**不是**「GTO 节点最多」，而是：**诚实（不编 EV/频率）＋ 离线零依赖 ＋ 个性化漏洞训练**。

### 商业模式：免费内核 + 一次性 Pro 解锁（首选，契合「诚实不套路」）

- **免费**（撑口碑、可自然传播）：翻前训练器 + 范围图表（13×13）+ 基础统计 + 错题复习。
- **Pro 解锁**（差异化能力）：🔍 个人画像 / 漏洞分析、🗓 训练计划、全部**自算 Nash 推弃档**（8–25bb / 6 人 / 单挑 HU / 面对全下）、算胜率计算器、将来的真数据导入。
- **计费**：数字商品走 StoreKit（iOS）/ Play Billing（安卓），建议用 **RevenueCat** 统一两端 + 校验授权；定价把平台抽成 **15–30%** 算进去。一次性买断 ~$5–15；想要经常性收入可叠加「**带云同步的订阅**」（但订阅必须持续有新价值，否则退订/差评）。
- **红线**：**绝不把现已免费的核心锁起来**——保住「诚实、可及」招牌；Pro 只锁 power features，不锁基础训练。
- 代码侧用一个 `isPro` 功能门控（先本地开关，接 RevenueCat 时换成真授权）。

## 2. 硬约束（任何新功能都必须遵守）

这些是产品的地基，不是临时限制：

1. **零构建、单页**：`gto-trainer.html` + `js/*.js` 普通 `<script>`，`file://` 双击即开。不引入打包器/框架/ES module。
2. **离线、无后端**：数据与进度只存 `localStorage`。不依赖服务器、账号、网络。**唯一例外**：Pro 的内购/授权校验（StoreKit / Play Billing / RevenueCat）需一次性联网，授权结果**本地缓存**——除此之外 app 全程离线，训练/数据不依赖网络。（订阅若做云同步则另算，须单独评估，见 §1 商业模式。）
3. **默认不接 LLM/API**：保持离线与零成本。若将来要「真 AI 教练」，须作为**独立架构决策**单独评估（会打破上面两条）。
4. **诚实第一**：
   - 范围是**手搓近似**，对外一律称「**参考范围**」，绝不冒充 solver 精确解。
   - **不显示编造的 EV 或精确混合频率**。EV/精确 % 只在**可计算**（推弃 all-in equity）或**导入真数据**的 spot 出现。
   - 每个 spot 带 `confidence` 标签（见 §6），在**场景层面**诚实，而非只有一句全局免责。
5. **回归测试护栏**：改 `MODES`/范围/引擎，先过 `npm test`；有意改动则 `npm run test:update` 复核快照 diff 再提交。

## 3. 现状（已落地）

- 训练器：开局/防守/面对 3bet·4bet/挤压/冷跟/推弃 等模式（`MODES` 单一来源驱动）。
- 范围图表（13×13 矩阵）、错题复习堆、生涯统计（`statsBySpot`）。
- **频率地基（Phase 1）**：`handFreq(t,hand)` → 每手动作权重；`freqTable` 可被真数据覆盖；`src/confidence` 标签；测试锁死「频率支持集 == 判定支持集」。
- **频率显示与评级（Phase 2/3）**：精确局面的混合点按真频率评级（最高频=最佳、次频=好棋；占位 50/50 仍并列「好棋」，不编造高下）；反馈与图表（`cInfo`）对精确局面显示真实百分比，其余维持定性。
- **自算 Nash 数据（Phase 7）**：`tools/` 下自建 all-in equity 引擎 + HU/多人 push/fold Nash 求解器；**8/10/12/15/20bb 9 人推弃 + 单挑 HU 10/15/20bb 各位置已用自算 Nash 替换手搓范围**（`confidence:'precise'`，数据在 `js/data/pushfold.js`/`hu-pushfold.js`，离线 `node tools/gen-pushfold.js` 重算）。
- **画像 / 训练计划（Phase 6）**：「统计」页「🎯 个人画像」(风格倾向松/紧 + 打法倾向被动/激进 + 强弱档位) + 「🗓 训练计划」(按需练度排序、一键去练)；纯本地聚合，诚实标注 vs 参考范围、不编造 TAG/LAG。错误分类已细分到 太松/太紧/被动/过激/边缘/ICM（`addMistake` 存真实选择，`classifyMiss` 据此精确分类）。
- **翻后胜率（带公共牌）**：「算胜率」计算器现支持填**牌面**（翻牌/转牌/河牌）算**翻后 equity**（`rangeEquityBoard`，纯枚举/蒙特卡洛，自动按组合加权 + 跳过冲突）；留空则仍是翻前全下。
- **个性化漏洞分析（Phase 5）**：「统计」页新增「🔍 你的漏洞」——从错题堆按类型（太松/太紧/边缘/ICM）聚合排序 + 最常踩的坑 + 一键去练（`classifyMiss` 仅凭 spot+hand 判定，旧数据也能分析）。
- **离线 HU 翻后 CFR+ solver 链**（`tools/solver/`，纯 Python）仍保留、由 `test/solver-spots.test.js` 验证，产物 `js/data/postflop-spots.js`（`python tools/gen-postflop-spots.py` 重算）。**App 内的「翻后GTO」展示页已移除**（产品决定）——离线链与数据留作管线/测试，不再进浏览器。
- 工程：多文件拆分、零依赖 Node 回归测试（契约不变量 + 金快照 + equity/Nash + 翻后数据/board-equity 验证，38 项）、git。

## 4. 架构分层（按我们的现实改写）

```
            Poker Preflop Coach
                    |
   1. Strategy Data   参考范围 + 频率（手搓/可导入真数据）
   2. Training        Range / Decision / Speed 训练
   3. Personal Coach  漏洞分析 + 错误分类 + 画像 + 训练计划   ← 核心差异化
   4. Math Tools      Range vs Range 胜率 + 推弃 EV（纯计算、零数据风险）
   5. Tournament      ICM（推弃可诚实算的子集）              ← 较后
```

外部 V3.0 的「AI Review / NL 解释」在我们这里由**模板化 `reasonFor()`** 承担（离线诚实替身），不做 LLM 版本，除非 §2.3 的架构决策改变。

## 6. 数据与诚实原则

**可信度三档**（每个 spot 的 `confidence` 字段）：

| 档位 | 含义 | 能否显示精确 EV/% |
|---|---|---|
| `precise` | 有 `freqTable` 真数据导入，或公开 Nash 解（如 10bb 推弃） | ✅ 可 |
| `curated` | 手搓、核对过量级/形状，混合频率是 `MIX=0.5` 占位 | ❌ 只显示「约/混合」，不给精确数 |
| `approx` | 最粗（ICM / 6 人估计） | ❌ 明确标「粗略估计」 |

**频率模型**：`handFreq` 优先用 `freqTable`（真数据），否则从手搓 R/C/M 派生占位权重。真数据导入即覆盖派生值并升级 `confidence:precise`。

**EV 红线**：除「推弃 all-in equity 可算」与「导入真数据」外，**不显示任何 EV 数字**。

## 7. 明确排除（写清楚，防跑偏）

| 排除项 | 原因 |
|---|---|
| **编造的精确 EV 数字**（如 Call: -0.16BB） | 我们无来源；对标榜诚实的工具是致命背叛。只在推弃/真数据处给。 |
| **编造的精确混合 %**（如 Fold 96/Call 3/Jam 1） | 派生的 50/50 假不出来；只有真数据或推弃才给真 %。 |
| **「AI 教练」LLM 自然语言引擎** | 要接 API/后端，打破离线+零构建+零成本（§2）。用模板 `reasonFor()` 替代。 |
| **商业 / 第三方版权 GTO 数据** | 版权与成本风险；坚持原创整理或公开可算数据。 |
| **浏览器内实时翻后 GTO 求解** | CFR 求解需离线 Python，浏览器跑不了任意输入。翻后在 App 内只做一件诚实的事：**翻后 equity**（纯数学，实时，填牌面即可）。离线 solver 真算的 GTO 范例仅作**离线产物/测试**（`js/data/postflop-spots.js`），**App 内展示页已移除**。**不做**自定义牌面/范围的实时策略求解，也不做完整翻后训练器——本项目主线仍是翻前。 |
